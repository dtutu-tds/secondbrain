# Technical Stack

## Platform

- **Obsidian Vault**: Local markdown-based knowledge base
- **Cross-platform**: Desktop (Windows/Mac/Linux) and Mobile (iOS/Android)

## Required Plugins

### Core Plugins
- Templates (built-in)
- Properties (built-in)

### Community Plugins
- **Templater**: Dynamic template generation with date/time functions
- **Dataview**: Query and aggregate data from notes
- **QuickAdd**: Automated note creation and capture workflows
- **Advanced URI** (optional): Deep linking to commands

## File Format

- Markdown (.md) files
- YAML frontmatter for structured metadata
- Inline properties using `key:: value` syntax
- Wikilinks for cross-referencing: `[[Note Name]]`

## Data Models

### Supported Note Types
- `daily`: Daily journal entries
- `weekly`: Weekly reviews
- `project`: Projects with tasks
- `area`: Areas of responsibility
- `goal`: Long-term goals
- `substance`: Biohacking substances/practices
- `literature`: Literature notes
- `idea`: Ideas and fleeting notes
- `note`: Default type

## Common Commands

### Creating Notes
- Use QuickAdd commands: `Daily_Lite`, `Daily_Full`, `Work_Block`, `New_Task`, `Bio_Stack`
- Access via Command Palette (Ctrl/Cmd+P) or mobile toolbar

### Template Usage
- Templates located in `90_System/Templates/`
- Use Templater syntax: `<% tp.date.now("YYYY-MM-DD") %>`
- Date formats: `YYYY-MM-DD`, `gggg-[W]ww` (week number), `DD-MM-YY`

### Querying Data
- Use Dataview queries in dashboard files
- Common patterns:
  ```dataview
  TABLE field1, field2
  FROM "folder"
  WHERE condition
  SORT field DESC
  ```

## File Naming Conventions

- Daily notes: `DD-MM-YY.md` (e.g., `15-11-25.md`)
- Weekly notes: Use week number format `YYYY-[W]WW`
- Projects/Goals: Descriptive names with spaces allowed
- Substances: Substance name as filename

## Configuration Files

- QuickAdd: `.obsidian/plugins/quickadd/data.json`
- Templater: `.obsidian/plugins/templater-obsidian/data.json`
- These are auto-generated by Obsidian plugins
